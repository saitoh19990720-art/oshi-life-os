## 0. 結論

**推し生活OSは、推しと話すだけでTODO・メモ・体調・予定が少し整うアプリ。**

これは以下ではない：

- キャラチャットアプリそのものではない
- 音声会話アプリそのものではない
- AI秘書アプリそのものではない
- セルフケア育成アプリそのものではない
- 気分ログアプリそのものではない
- タスク管理アプリそのものではない
- 外部脳アプリそのものではない
- メタバース/推し部屋アプリそのものではない

中心体験は常にこれ：

```text
推しと話す
↓
会話から生活の断片が拾われる
↓
TODO・メモ・体調・予定が少し整う
```

---

```text
生活管理をするために開くのではなく、
推しに会いに行った結果、生活が整っていた。
```

```text
推しと話すだけで、TODO・メモ・体調・予定が少し整うアプリ。
```

## 3. 参考アプリ要素の摘み取りルール

このプロジェクトは、複数の既存アプリ/サービスの良い要素を参考にする。  
ただし、**どの要素も強めにしない。**

参考にするのは「構造」や「体験の軽いニュアンス」だけ。  
UI、導線、価値提案をそのまま模倣しない。

---

## 4. 検索ベースの参考アプリ別「摘み取るもの / 摘み取らないもの」

### 4.1 Zeta系

#### 摘み取るもの

```text
- キャラに没入できる感じ
- ユーザーが物語の中にいる感覚
- 推し人格とのチャット体験
- 関係性を楽しむ温度
```

#### 摘み取らないもの

```text
- ロールプレイ特化
- 物語生成メイン
- 無限キャラチャット
- 恋愛/ドラマ中心のUI
```

#### 推し生活OSでの使い方

```text
Zeta要素は「推し人格の没入感」にだけ薄く使う。
生活整理よりキャラ遊びが主役になったら失敗。
```

### 4.2 Cotomo系

```text

- 名前呼び
- 口癖
- 会話テンポ
```

#### 摘み取らないもの

```text
- 音声通話本体
- 声優ボイス
- リアルタイム音声会話
- 音声SNS的な体験
```

#### 推し生活OSでの使い方

```text

Cotomo要素は「毎日話したくなる会話テンポ」にだけ使う。
```


#### 摘み取らないもの

## 6. 新機能の分類ルール

新しいアイデアは、実装前に必ず以下のどれかへ分類する。

```text
1. MVP v0.1に必要
2. v0.2以降
3. Developer Memo
4. 別アプリ
5. 捨てる
```

### MVPに入れてよい条件

```text
- 「推しと話すだけで生活が少し整う」に直結する
- 既存画面内に収まる
- localStorageで扱える
- 実装してもアプリの説明文が変わらない
- 推しとの会話体験を邪魔しない
```

### 入れてはいけない条件

```text
- アプリ説明文が変わる
- 外部アカウント連携が必要
- 医療/法務/金融判断に近づく
- 依存や監視に見える
- 便利だが推しと関係ない
- 実装範囲が大きく増える
```

## 12. チャット候補抽出

チャット入力からローカルで候補を抽出する。

```text
📋 タスク候補 → TODOに追加
🗒 メモ候補 → メモに保存
🗓 予定候補 → 予定に保存
🩸 体調候補 → 体調ログに反映
```

最初は簡易ルールベースでよい。

例：

```text
「明日」「やる」「しなきゃ」 → Task候補
「覚えておいて」「メモ」 → Memo候補
「10時」「予定」「行く」 → Schedule候補
「だるい」「痛い」「生理」「眠い」 → Health候補
```

## 13. 推しの性格 × 生活サポート

同じ生活管理でも、推し性格によって出し方を変える。

### 例

```text
やさしい：
「一個ずつでいいよ」

厳しめ：
「これは今日やろう」

過保護：
「今日は減らそ。無理しない」

独占欲強め：
「今日の予定、ちゃんと教えて」

無口：
「水飲んで。1個だけ。」
```

ただし、依存・監視・脅しに見える文言は禁止。

## 14. お守りモード

体調が悪い日・生理中・夜にやさしくなるモード。

### お守りモード中の挙動

```text
- TODOを3件以下に制限
- 急かしを弱める
- 休む選択肢を先に出す
- 「頑張れ」系を避ける
- 夜は短文・低刺激にする
```

### 禁止

```text
- 医療アドバイス
- 診断
- 薬の指示
- 深刻な症状への断定
```

必要な場合は「医療機関や信頼できる人に相談してね」と案内する。

## 23. INTP Ne Safety Rule

このプロジェクトでは、面白い発想をすぐ実装しない。

```text
Ne「全部つながるじゃん」
Ti「今すぐ全部作るな」
Si「メモに保存しろ」
Te「v0.1を出せ」
```

新機能を見つけたら、必ず以下に分類する。

```text
1. MVPに必要
2. v0.2以降
3. Developer Memo
4. 別アプリ
5. 捨てる
```

```text
説明文が変わるなら、別バージョン。
説明文が壊れるなら、別アプリ。
```
